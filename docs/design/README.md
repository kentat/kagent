## ドキュメント更新ルール

1. **設計変更時は必ず更新する**

   各文書の更新履歴セクションに日付・変更内容を追記

3. **LESSONS.mdはバグ修正のたびに追記する**
   コードを直したら「なぜ起きたか・対策は何か」を記録する
