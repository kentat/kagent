## 1. エージェント定義の構造

### 3層構造

```
Layer 1: COMPANY.md
  └─ すべてのエージェントが読む「会社の憲法」
  └─ 会社の使命・オーナー情報・共通ルール

Layer 2: {NAME}.md（個別職務定義書）
  └─ そのエージェントだけの職務・権限・出力ルール

Layer 3: LESSONS.md
  └─ 過去のミスと対策の記録
  └─ 継続的改善の源泉
```

### agent.py での読み込み

```python
def _load_agent_def(name: str) -> str:
    company = read_md("COMPANY.md")   # 全員共通
    lessons = read_md("LESSONS.md")   # 学習記録
    agent   = read_md(f"{name}.md")   # 個別定義

    # 3つを結合してシステムプロンプトとして使用
    return f"""
# 会社全体のルール
{company}
---
# 学習記録（過去のミスと対策）
{lessons}
---
# あなたの職務定義
{agent}
"""
```

## 3. 現在の3エージェント定義サマリー

### AOI（アオイ）- 秘書エージェント

| 項目 | 内容 |
|------|------|
| 役割 | 唯一の窓口・オーケストレーター |
| ツール | なし |
| インプット | けんたのメッセージ |
| アウトプット | SORAへのタスク定義文字列 |
| 禁止事項 | データ収集・出力整形 |

### SORA（ソラ）- 分析エージェント

| 項目 | 内容 |
|------|------|
| 役割 | データ収集・分析の専門家 |
| ツール | 19種類すべて |
| インプット | AOIからのタスク定義 |
| アウトプット | 生データ（整形なし） |
| 禁止事項 | 出力の整形・けんたとの直接会話 |

### RIO（リオ）- デザインエージェント

| 項目 | 内容 |
|------|------|
| 役割 | 出力整形・読みやすさの専門家 |
| ツール | なし |
| インプット | SORAからの生データ |
| アウトプット | Telegram向けフォーマット済みテキスト |
| 禁止事項 | データ収集・データの改ざん |

## 4. システムプロンプト設計ガイドライン

### 書くべきこと

| カテゴリ | 内容 | 例 |
|---------|------|-----|
| アイデンティティ | 名前・役割・性格 | 「あなたはSORAです」 |
| 行動ルール | やること/やらないこと | 「数字は省略しない」 |
| 出力形式 | 具体的なサンプル付き | 「✅ 良い例: ...」 |
| コンテキスト | オーナー情報など | COMPANY.md参照 |
| 学習記録 | 過去のミスと対策 | LESSONS.md参照 |

### 書いてはいけないこと

| 内容 | 理由 |
|------|------|
| 毎回変わるデータ | ユーザーメッセージで渡す |
| 会話履歴 | storage.pyで管理 |
| 重複情報 | COMPANY.mdに書いたことを再度書かない |
| 曖昧な指示 | 「丁寧に」より「敬語を使わず要点から入る」 |

### トークン管理

```
理想的なサイズ目安：
  COMPANY.md : ~500トークン（共通・簡潔に）
  {NAME}.md  : ~300トークン（役割に集中）
  LESSONS.md : ~200トークン（重要なものだけ）
  合計        : ~1000トークン以下を目標

300行ルール（業界経験則）:
  300行を超えるとClaude APIの挙動が不安定になる場合がある
  不要な行は削除：「これがなければミスするか？」でフィルタリング
```

## 5. Markdown記述ルール

### 効果的な記述

```markdown
# 良い例
### ✅ やること
1. **ツールを積極的に使って**情報を収集する
2. 数字は**省略・丸めずに正確に**出力する

# NG例と良い例のセット（特に効果的）
✅ 良い例: +¥195,296（+$1,247）+7.98%
❌ 悪い例: +1247ドル
```

### 避けるべき記述

```markdown
# 悪い例
- 丁寧に対応する（曖昧・測定不可能）
- わかりやすく出力する（何が「わかりやすい」か不明）
- 正確に答える（どの程度の正確さか不明）
```
