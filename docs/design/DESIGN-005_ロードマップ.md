## Phase 4: 完全分離（🚀 長期目標）

**目標**: 2027年〜  
**テーマ**: マイクロサービス化・MCP対応

### アーキテクチャ変更

```
現在: 1プロセス（Railway 1サービス）
  AOI + SORA + RIO が同一プロセス

将来: 3プロセス（Railway 3サービス）
  AOI Service   → Port 8001
  SORA Service  → Port 8002  
  RIO Service   → Port 8003
  
  通信: HTTP REST または Redis Queue（A2A対応）
  共有状態: Redis
```

### 技術的マイルストーン

- [ ] Redis完全移行（Phase 2で達成）
- [ ] SORA を FastAPI エンドポイントとして分離
- [ ] RIO を FastAPI エンドポイントとして分離
- [ ] tools.py を MCPサーバーとして公開
- [ ] A2A（Agent-to-Agent Protocol）対応

### 目標コスト

| フェーズ | Railway費用 | 説明 |
|---------|------------|------|
| 現在 | ~$10/月 | 1サービス |
| Phase 4 | ~$20/月 | 3サービス + Redis |

## 機能追加の判断基準

新機能を追加する前に以下を確認：

```
1. 現在のシステムは安定しているか？
   → 不安定なら機能追加より安定化を優先

2. Redis移行の設計に反していないか？
   → 新しい状態はstorage.py経由で保存する

3. どのエージェントの担当か？
   → ツールが必要 → SORAに追加
   → 出力整形変更 → RIO/schedulerを修正
   → 指示変更 → AOI/schedulerを修正

4. LESSONS.mdに追記が必要か？
   → バグ修正後は必ずLESSONS.mdを更新する
```
